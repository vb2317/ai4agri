Hi,

I’m sharing the current UPAHAR Section 12 demo UI for a design review. It is a local prototype of a map-first agriculture intelligence dashboard tied to an RFP.

The easiest way to open it on a Mac is to double-click:

```text
Start UPAHAR Demo.command
```

If that does not work, double-click:

```text
Open Static Preview.command
```

The main context is in:

- `handoff/designer/README.md`
- `handoff/designer/design_brief.md`
- `handoff/designer/rfp_brief.md`
- `rfp/section12.md`

The current UI shows satellite imagery, parcel boundaries, crop classification, parcel confidence, stress/NDVI signals, field evidence, procurement planning, RFP traceability, and a SAM-assisted boundary workflow. Some data is demo/mock data, and the UI labels should make that clear.

The useful review lens is whether the first screen feels credible, understandable, and suitable for a government/tender-review conversation.
